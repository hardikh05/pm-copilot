# Sample data

Two kinds of files in this folder.

## Inputs — the kind of data PM Copilot ingests

| File | Contents | Use for |
|---|---|---|
| [`sample_feedback_50.csv`](./sample_feedback_50.csv) | 50 synthetic recipe-app feedback items | Quick test — small enough to inspect by eye |
| [`sample_feedback_500.csv`](./sample_feedback_500.csv) | 500 synthetic items across 8 themes | The "real" demo dataset |
| [`sample_feedback.json`](./sample_feedback.json) | Same 500 items, JSON variant | Demonstrating the JSON upload path |

All three are generated by `backend/app/synthetic/recipe_app.py` with `seed=42`, so they're reproducible.

**Columns:** `text`, `source`, `rating`, `user_segment`, `author`, `created_at`. Only `text` is required — everything else is auto-extracted by the ingestion parser (`backend/app/services/ingestion.py`), which is tolerant of column-name variations (`text`/`body`/`comment`/`review` all work).

**Try it:** in the UI → Project → Add feedback → drag any of these files in.

### Themes seeded in the data

The synthetic generator stuffs feedback into these 8 themes (with sentiment weights that make clusters look realistic):

1. Payment failures (20%, mostly negative)
2. Recipe generation quality (18%, mixed)
3. Search bugs (14%, negative)
4. Onboarding friction (12%, negative)
5. Praise for AI suggestions (12%, positive)
6. Cross-device sync issues (10%, negative)
7. Missing dark mode (7%, mild requests)
8. Pricing complaints (7%, negative)

A successful agent run should rediscover roughly these 8 clusters.

## Outputs — what the agents actually produced

These are real snapshots from a live pipeline run on the seeded 500-item dataset. Useful for showing recruiters "here's what the system produces" without needing them to run the live demo.

| File | Contents |
|---|---|
| [`sample_output_clusters.json`](./sample_output_clusters.json) | Discovered clusters with labels, summaries, sentiment breakdown, and source feedback IDs (traceability) |
| [`sample_output_features.json`](./sample_output_features.json) | Features derived from clusters, ranked by RICE, with full score breakdown and strategy bucket |
| [`sample_output_roadmap.json`](./sample_output_roadmap.json) | Now / Next / Later placement with themes and dependencies |
| [`sample_output_prds.json`](./sample_output_prds.json) | One full PRD (problem, goal, stories, metrics, requirements, edge cases, out of scope) plus Critic notes from the second-pass review |

The PRD sample is a representative artifact — same JSON shape as `backend/app/agents/prd_generator.py` produces, critique notes match `backend/app/agents/prd_critic.py`. The `_meta.note` field on the PRD calls this out explicitly.

## Regenerating

```bash
python backend/scripts/export_demo_data.py [project_id]
```

Inputs always regenerate from the same seed (deterministic). Outputs snapshot whichever project ID you pass (default: 5). Run it after a fresh agent pipeline to refresh the output snapshots.
